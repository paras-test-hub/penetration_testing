Web Application Penetration Test Report

Target: coach.mastersunion.org | Date: September 13, 2025 | Tester: Paras / Team

Executive Summary

A focused penetration test was performed against coach.mastersunion.org. The assessment combined authenticated and unauthenticated scans (Nmap, Nikto) and manual checks for common web vulnerabilities. This report outlines the highest priority findings, provides a concise risk summary, and recommends next steps for remediation.

Findings & Recommendations

HIGH

Sensitive Data in JWT Token

An authenticated Nmap scan revealed that the JWT token, used for session authentication, contains sensitive Personally Identifiable Information (PII) such as the user's name, email, and mobile number in plain text. This data is not encrypted and is easily readable, posing a significant data leakage risk if the token is intercepted.

Recommendation:

The JWT token payload should not contain sensitive PII. Remove this information from the token and instead use a non-descriptive identifier (e.g., a UUID) to reference user data on the server-side.

MEDIUM

Missing HTTP Security Headers

Nikto and Nmap scans identified that the website is missing key security headers. The X-Frame-Options header, which prevents clickjacking attacks, is not present. Additionally, the Strict-Transport-Security (HSTS) header is not defined.

Recommendation:

Implement the following headers to improve security:

  • X-Frame-Options: SAMEORIGIN to prevent clickjacking.
  • Strict-Transport-Security to enforce HTTPS and protect against man-in-the-middle attacks.
  • X-Content-Type-Options: nosniff to prevent MIME type sniffing.

MEDIUM

Sensitive Files Exposed

The Nikto scan found sensitive file exposures, specifically an .htpasswd file which could contain user credentials or hashes. This file should not be publicly accessible.

Recommendation:

Remove the exposed sensitive file. Ensure that web server configurations prevent direct access to such files in the future.

LOW

Open Ports

Nmap scan of all ports on the target host found that ports 21 (FTP), 80 (HTTP), and 443 (HTTPS) are open. This exposes additional services that could be potential attack vectors.

Recommendation:

Close any unnecessary ports. The FTP service on port 21 is not recommended for a public-facing website and should be closed if not required. Harden the security of all open ports by applying the latest patches and secure configurations.

Conclusion

The target coach.mastersunion.org exhibits several security vulnerabilities across authentication, HTTP security headers, and network exposure. The most critical finding is the leakage of sensitive PII within the JWT token. Recommendations have been provided to mitigate the identified risks. It is strongly advised to prioritize the high-severity fixes and re-scan the application after remediation to validate the changes.
