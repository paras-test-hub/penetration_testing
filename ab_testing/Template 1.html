Apache Bench Report: 50k Requests Analysis

Benchmarking results for api.mastersunion.in across various concurrency levels.

Executive Summary

This report summarizes the performance of the api.mastersunion.in server under three different load conditions: 100, 500, and 1000 concurrent users. The tests were performed using ApacheBench with 50,000 requests for each scenario. The results show that the server performs optimally at a lower concurrency level, with performance degrading significantly as the number of concurrent users increases beyond 500. Key Takeaways

  • Optimal Performance at 100 Users: The server handles 100 concurrent users with a low failure rate and quick response times.
  • Increased Load, Increased Latency: Moving from 100 to 500 users, the server's overall throughput (requests/sec) increases, but this is accompanied by a significant rise in latency and failed requests.
  • Performance Degradation at 1000 Users: At 1000 users, the server is clearly overloaded. Requests per second drop by almost half compared to the 500-user test, and the median response time increases by over 300%.
  • Longest Request Spikes: The maximum time for a single request jumps from 1418 ms (100 users) to over 70 seconds (1000 users), a strong indicator of severe performance issues under heavy load. [cite: 19, 26]
