What the AB report contains (explained)
Apache Bench output includes several important sections — here is how to read them and what they tell you about the system:
  • Concurrency Level — how many simultaneous requests were made. Higher concurrency stresses connection handling and thread pools.
  • Time taken for tests — total wall-clock time to finish all requests; used with total requests to compute throughput.
  • Complete requests / Failed requests — success vs failure count. Failed requests indicate server errors, timeouts, or network problems.
  • Requests per second (RPS) — average throughput. Useful to compare across runs.
  • Time per request (mean) — average latency (both per concurrent client and across all requests).
  • Percentage served within X ms — latency distribution (50%, 90%, 95%, 99% percentiles) — crucial for understanding tail latency experienced by users.
  • Transfer rate — data throughput in Kbytes/sec. Shows bandwidth usage.
  • Socket errors — e.g. connect errors or timeouts; often indicate saturation or network issues.
Interpreting common results
  1. High RPS with low latency — service handles load well.
  2. Rising latency with concurrency — queueing at app server or DB; investigate thread pool and DB connection limits.
  3. Many failed requests — check server logs for 5xx errors, timeouts, or rate limiting.
  4. High p95/p99 — outliers likely caused by GC pauses, slow DB queries, or background jobs blocking request threads.
